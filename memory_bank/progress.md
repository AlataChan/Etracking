## [2025-05-09] 会话管理模块修复

### 问题描述
在会话管理器(`session_manager.py`)的`_select_company_role_simple`方法中发现JavaScript语法错误：`SyntaxError: missing ) after argument list`。这个问题出现在处理泰国海关网站上的公司角色选择功能时，选择器字符串中包含单引号，当嵌入到JavaScript中也使用单引号包围时导致语法解析错误。

### 解决方案
1. 将所有JavaScript评估(`evaluate`)方法中的代码改为使用三重引号和双引号：
   ```python
   # 错误写法
   self.page.evaluate(f"document.querySelector('{input_selector}').checked")
   
   # 正确写法
   self.page.evaluate(f"""
       document.querySelector("{input_selector}").checked
   """)
   ```

2. 简化公司角色选择逻辑，采用多种方法尝试选择，包括：
   - 直接点击input元素
   - 使用JavaScript强制设置checked属性
   - 通过标签文本查找并点击label元素

3. 对于进口商角色，采用验证状态而不是强制选择的策略，减少出错可能

### 重要经验教训
1. **特殊字符处理**：在处理包含引号的选择器时，必须注意JavaScript字符串的引号嵌套问题
2. **多行字符串**：对于复杂的JavaScript代码，使用Python的三重引号格式可以提高可读性和避免字符转义问题
3. **多层错误处理**：实现多种方法尝试完成同一操作，即使某些方法失败也能继续执行
4. **状态验证**：每次操作后验证状态，确保操作真正生效
5. **降级策略**：在非关键路径上采用警告而非错误，避免整个流程因小问题而中断
